# design.md

## 対象

`renewal/` 配下に新規構築するクロヤナギ医院サイト。

## 目的

既存サイトと `clinic/` の作りかけサイトを温存しながら、公開切り替え前提の新しい医院サイトを独立して作る。初期段階ではトップページの骨格、情報設計、デザイン基調を固め、今後の詳細調整に耐えられる構成にする。

## 実装方針

### 1. ディレクトリ分離

- 新サイトは `renewal/` に閉じて実装する。
- 既存ルートサイトと `clinic/` は参照対象として残し、完成前に置き換えない。
- 公開切り替え時は、`renewal/` の成果物を公開対象にするか、ルートへ移す方針を別途決める。

### 2. 技術構成

- Next.js App Router を使う。
- 依存関係は既存 `clinic/` と同等の軽量構成から始める。
- CSS は初期段階では `app/globals.css` に集約し、サイト全体のトーンを一箇所で調整できるようにする。
- 画像は `renewal/public/photo/` に配置し、Next.js 開発環境で自己完結して確認できるようにする。

### 3. 情報設計

- ファーストビューでは医院名、診療科目、電話番号、受付につながる主要情報をすぐ見せる。
- 診療時間とアクセスはトップページ内で確認できる位置に置く。
- お知らせは初期表示では静的データとして扱い、CMS 連携は後続検討とする。
- 初めての方へは、受付時に必要なものや来院前の不安を下げる内容を置く。

### 4. デザイン方針

- 白を基調に、淡いブルーグリーンと落ち着いた紺をアクセントにする。
- 医療機関らしい清潔感を優先し、装飾は控えめにする。
- 角丸や影は弱めにし、情報の読みやすさを優先する。
- 既存写真素材を使い、実在感のある医院サイトにする。

### 5. レスポンシブ方針

- PC ではヘッダー、ファーストビュー、診療案内を横方向に整理する。
- スマートフォンでは診療時間、電話、アクセスへの導線を優先して縦積みにする。
- ボタンや電話番号はタップしやすいサイズを維持する。

## 初期変更ファイル

- `renewal/package.json`
- `renewal/next.config.mjs`
- `renewal/app/layout.js`
- `renewal/app/page.js`
- `renewal/app/globals.css`
- `renewal/public/photo/`

## 検証方針

- `renewal/` で `npm install` を実行できること。
- `renewal/` で `npm run build` が成功すること。
- 必要に応じて `npm run dev` でローカル表示を確認すること。
- 既存ルートサイトと `clinic/` に不要な変更が入っていないこと。
