# design.md

## 対象

`clinic/` 配下の「クロヤナギ医院」トップページ改修。

## 目的

添付されたデザインモックを基準に、医院サイトとして信頼感のあるトップページを Next.js で再構成する。特に、ヘッダー、ファーストビュー、診療案内セクションの再現度と、広幅時に崩れないレイアウト安定性を重視する。

## 実装方針

### 1. レイアウト構造

- 背景を全幅、内部コンテンツを最大幅制御する二層構造を基本とする。
- 全幅で見せたい領域:
  - ヘッダー背景
  - ファーストビュー背景画像
- 一定幅に収める領域:
  - ヘッダー内部要素
  - ファーストビューのテキスト・CTA・補助カード
  - 診療案内以下の各セクション

### 2. ヘッダー

- 白背景の帯を全幅で敷き、その中にロゴ、ナビ、電話を中央配置する。
- ロゴ、ナビ、電話の 3 要素が広幅で分離しすぎないように、内部コンテナで最大幅を制御する。
- 電話番号は右寄せ、ナビは中央、ブランドは左寄せを基本にする。

### 3. ファーストビュー

- 背景は 1 枚の画像を全面に使用する。
- 左側にのみ白〜淡青のオーバーレイを重ね、コピーの可読性を確保する。
- テキストと画像を別カラムとして分離しない。
- CTA は横並び 2 ボタンを基本とし、狭幅では縦積みに落とす。

### 4. 診療案内セクション

- `診療案内` を親セクションとして新設する。
- 構成:
  - セクション見出し
  - 2 カラムカード (`診療科目` / `診療時間`)
  - 下段カード (`ご来院の方へ`)
- `診療科目` は代表科目を大きく見せ、補助的に他科目も表示する。
- `診療時間` は表を主役にし、下部に休診日と受付注記を並べる。
- `ご来院の方へ` は上段 2 カラムと視覚的に整合する横幅基準を持たせる。

### 5. タイポグラフィ

- ファーストビューより下の見出しは、FV より強くなりすぎないように抑制する。
- 明朝系の見出しとゴシック系本文のコントラストを維持する。
- セクションごとの文字サイズを統一ルールで管理し、局所的に大きくしすぎない。

### 6. アイコン

- インライン SVG を利用する。
- 円背景付きアイコンでは、背景色と線色の競合が起きないよう、必要な箇所に明示的な色指定を与える。
- 既存 UI と同じ線画トーンで統一する。

### 7. レスポンシブ対応

- 広幅時:
  - 内部コンテンツ幅を固定し、背景だけを広げる
  - 行長が伸びすぎないよう制御する
- 中幅時:
  - ヘッダー要素を段組みに逃がす
  - 診療案内 2 カラムの余白と表サイズを調整する
- 狭幅時:
  - CTA やカードを縦積みにする
  - 診療案内カード内の要素を中央寄せに再配置する

## 変更ファイル

- `clinic/app/ClinicHomePage.js`
- `clinic/app/globals.css`
- 必要に応じて `clinic/app/layout.js` または関連設定ファイル

## 検証方針

- `npm run build` が成功することを確認する。
- `npm run dev` でローカルプレビューできることを確認する。
- 広幅表示でヘッダー、FV、診療案内セクションが崩れないことを確認する。
- 診療案内セクションで、上段 2 カラムと下段カードの幅関係が意図どおりであることを確認する。
