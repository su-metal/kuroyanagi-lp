# requirements.md

## 対象作業

`clinic/` 配下の Next.js サイトとして実装している「クロヤナギ医院」トップページの改修。

## 要求内容

- 添付デザインを基準として、クロヤナギ医院サイトのトップページを再現する。
- 既存の LP 的な情報配置ではなく、医院サイトとして自然な情報整理と導線を持つ構成にする。
- `clinic/` は通常の Next.js 開発フローで確認できる状態を維持する。

## 変更対象

- ヘッダー
- ファーストビュー
- 診療案内セクション
- お知らせ
- アクセス
- みっかび東介護老人保健施設導線
- フッター
- レイアウト幅とレスポンシブ挙動

## ユーザーストーリー

- 利用者として、医院名・電話番号・診療時間・アクセスを最初の数スクロールで把握したい。
- 来院を検討している人として、診療科目や診療時間を見やすい形で確認したい。
- 介護施設との連携を知りたい人として、みっかび東介護老人保健施設への導線を自然に見つけたい。
- 開発者として、`npm run dev` でローカル確認できる安定した構成を保ちたい。

## 受け入れ条件

- `clinic/` で `npm run dev` によりローカル起動できる。
- トップページは添付デザインに近い構成、余白、色、情報階層を持つ。
- ヘッダーは広幅でも崩れず、内部コンテンツ幅は一定に保たれる。
- ファーストビューは 1 枚の背景画像と左オーバーレイで構成され、左右分離レイアウトにならない。
- 診療案内セクションは独立した親セクションとして構成される。
- `診療科目` と `診療時間` は 2 カラムで並び、下段の `ご来院の方へ` は意図した幅基準で揃う。
- 画像参照やレイアウト調整による変更後も `npm run build` が成功する。

## 制約事項

- 実装基盤は `clinic/` 配下の Next.js を維持する。
- 共有可能な画像素材は既存の `public/photo` を利用する。
- 画像そのものの完全一致ではなく、レイアウトと UI 構造の再現を優先する。
- 文言は医院サイトとして自然な日本語を維持する。
- ドキュメントは日本語で記述する。
