# requirements.md

## 対象作業

提示されたデザインリファレンス（添付画像）に基づき、`clinic/` 配下のクロヤナギ医院サイトのFV（ヘッダーおよびヒーローセクション）を刷新する。さらに、FeatureセクションをSticky Sidebarレイアウトにリデザインし、エディトリアルな読後感を向上させる。

追加作業として、ルートのLP問い合わせフォーム (`contact.php`) のメール送信がGmailの送信者認証要件に抵触しにくい固定From方式になっていることを維持する。

## 要求内容

- **デザインの忠実な再現**: 添付された最新のデザイン画像のレイアウト、装飾（縦書き、角丸、イラスト、あしらい）、余白感を正確に再現する。
- **ヘッダーの高度化**: 3つのアクションカード（予約・施設案内・電話）を持つ2段構成のヘッダーを実装。
- **ヒーローセクションの装飾**:
  - メイン写真の大きな角丸（独特の切り抜き）の再現。
  - PC幅ではメイン写真の左端をviewport左側に固定し、右端だけがウィンドウ幅に追従して伸縮する。
  - 縦書きのメッセージ「地域の皆さまの健康を支え、安心の毎日を。」の実装。
  - 縦書きメッセージはPC幅でviewport左側から固定オフセットに配置し、ヒーローイメージにオーバーラップさせる。
  - 家族イラスト、鳥、風景シルエット（家・橋・波）の配置。
  - 左下の風景イラストはPC幅でviewport左下から固定オフセットに配置し、ウィンドウ幅に合わせて伸縮させない。
  - 清潔感のある診療時間表オーバーレイ的配置。
  - 診療受付時間カードはPC幅でviewport右下から固定オフセットに配置する。
- **広域背景デザイン**:
  - Service, Facility, News の3つのセクションを跨いで表示される巨大な半円背景（.sections-with-bg-arc）を実装し、エディトリアルな連続感を演出する。
  - PC表示では、Facilityセクション内の右側にプライマリーカラーパネルを配置し、ServiceカードやNewsカードへ食い込まない安定した背景装飾として扱う。
  - モバイル表示では、Service, Facility, News を跨ぐ半円背景を共有ラッパー側の淡い青の背景レイヤーとして表示し、濃い青の半透明楕円に見えないようにする。
  - モバイル表示の Service と Facility の間には、波線と小鳥のあしらいを配置し、セクション間の余白を自然につなぐ。
- **FEATUREセクションのスクロール演出**:
  - PC幅では左側の縦書きコピーをsticky表示し、番号付きタイムラインを現在の特徴カードに連動させる。
  - 右側は番号なしの導入カードから始め、その後に4枚の特徴カードをスクロールで1枚ずつ表示する。カードは前のカードより少し下の位置で固定されながら重なる。
  - 1024px未満ではstickyと重なり演出を解除し、通常の縦積みカードとして読みやすく表示する。
- **ABOUTセクションの統一感**:
  - ABOUTセクションはオレンジやグレー寄りの印象を抑え、プライマリーブルー系のラベル、背景、装飾トーンへ寄せてサイト全体と統一する。
  - ABOUT背景は円が広がる前の初期状態をFVと同じ白背景にし、円形リビールはプライマリーカラーのベタ塗りを土台にする。薄い白の大きな楕円シェイプと小さな楕円シェイプは、円が開き切った後に表示し、ゆっくり動く装飾として扱う。下部イラストは既存イラストを使用し、素材感を残しながらごく弱いトーン調整だけをかける。
  - FVとABOUT初期背景は共通CSS変数で管理する。境界にグラデーションは使わない。
- **Vanilla CSS の使用**: フレームワークに頼らず、細部までコントロールされたスタイリングを行う。
- **問い合わせフォームのメール送信方針**:
  - PHP標準の `mb_send_mail()` 送信を維持する。
  - FromとEnvelope senderは `postmaster@kuroyanagi-clinic.jp`、送信先は `web@kuroyanagi-clinic.jp`、Reply-Toはフォーム入力者メールを維持する。
  - フォーム入力者メールをFromに使わない。

- 各要素の配置とデザインが最新の添付画像（写真ベース）と一致している。
- PC幅ではFVの主要要素が中央寄せで横移動せず、viewport各辺基準の固定オフセットで配置されている。
- 診療・サービスセクションが以下の構成を全て含んでいること：
  - イラスト付きの導入ヘッダー。
  - 左右交互レイアウトの「一般診療」と「外科手術」セクション。
  - 円形アイコンがオーバーラップした「その他のサービス」カード（3枚）。
  - 最下部のアクセシビリティ・アイコン列（4項目）。
- `clinic/` 配下で正常に動作し、既存のルートファイルに影響を与えない。
- レスポンシブ対応が行われている（PCデザインを優先しつつ、モバイルでも破綻しない）。
- モバイル版のヒーローエリアにおいて、メイン画像（スライダー）を高く配置し、左右の幅を広げて視覚的インパクトを強化する。
- ヒーローのキャッチコピーはPC版と同様に縦書き（vertical-rl）とし、各行に白帯を敷いて視認性を確保する。
- 配置は被写体を避けるため、中央から左側にオフセットして配置する。
- PC表示のFacility背景パネルは、Facility本文の白文字を読みやすく保ち、Serviceカードや注記、Newsカードの可読性を損なわない。
- ABOUTセクションはプライマリーカラーの印象に近づき、既存の読みやすさとスクロール演出を損なわない。
- ABOUTの円形リビール前はFVと同じ背景色でつながり、広がる円のプライマリーカラー面ではABOUTテキストを白系にし、薄い白シェイプと競合しないよう文字影で可読性を保つ。下部イラストはごく弱いトーン調整で本文を邪魔しない配置にする。
- ABOUT下部イラストは全幅表示を維持し、どの横幅でも雲や山などイラスト上部がABOUT背景の背面に潜り込んで欠けないよう、青背景より前面・本文より背面の重なり順で表示する。PCでは画像ボックスの高さを固定範囲に収め、イラスト内容は上基準で表示し、超ワイド幅では `bottom` 位置で調整する。
- FV下端とABOUT上端は同じCSS変数の白背景でつながり、グラデーションなしでも不自然な段差に見えない。
- モバイル表示の Service, Facility, News 背景では、淡い青の大きな半円がセクション背面に自然につながり、本文・写真・ボタンの可読性を損なわない。
- モバイル表示の Service と Facility の間のあしらいは本文やカードに重ならず、導線として控えめに見える。
- 問い合わせフォームの送信処理は送信失敗時に既存と同じエラー画面を表示する。
- 通知メールのFromは `kuroyanagi-clinic.jp` ドメインの固定アドレスになり、フォーム入力者メールはReply-Toにのみ設定される。
- **Wave Divider の実装**:
  - `news` セクションと `access` セクションの間に、`--bg-warm` (#e8eef2) を基調とした波型の区切り線を実装する。
  - `news` セクションの底辺が波型に見えるようにし、下側の `access` セクション（白背景）へ自然に繋がるようにする。
